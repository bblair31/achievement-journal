With the previous sort, when users have identical scores, it causes issues with pagination. **Across different page offsets (i.e. 10 per page) BigQuery will return an arbitrary order if you don't give it any unique sort keys**. So it could return the same user(s) across two different pages.

What I'm adding here is tertiary sorts by `last_activity_date` and `user_id` to guarantee uniqueness and predictable pagination
